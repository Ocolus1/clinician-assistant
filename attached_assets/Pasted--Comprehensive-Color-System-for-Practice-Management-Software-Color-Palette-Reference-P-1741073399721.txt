## Implementation Notes for Key Pages

### Dashboard Page
- Use white backgrounds for all cards and containers
- Use Primary Blue 500 only for icons, numbers, and interactive elements
- Keep the main content area background light gray (Gray 50)
- Use sparse and deliberate blue accents rather than large blue areas

### Sessions Page (CRITICAL FIX)
- Replace the solid blue background with white (#FFFFFF) for session cards
- Use a small colored left border or tag to indicate session status
- Maintain high contrast with dark text on light backgrounds
- Use blue highlights only for status indicators and interactive elements

### Create Session Form (CRITICAL FIX)
- All form sections should have white backgrounds
- Use tab design with subtle blue indicators for active state
- Maintain consistent form field styling
- Use blue only for buttons, active states, and accents

## Special Considerations

### Accessibility
- Ensure all text colors maintain at least 4.5:1 contrast ratio with their backgrounds
- Use color AND shape/text to convey status information
- Make sure interactive elements have clear hover/focus states

### Mobile Responsiveness
- Maintain color consistency across screen sizes
- Consider using slightly larger touch targets with the same color scheme on mobile
